nMiners

A DSL that aims to help the development of Data Mining Algorithms in parallel.

Introduction

  nMiners is a tool that can assist data scientists to manipulate big data in order to implement and run data mining algorithms in parallel.

  The tool is a DSL that was built using Scala language. The DSL code will be translated to a Scala code that uses Apache Mahout Library’s algorithms to process big data. Apache Mahout is a library that provides a set of data mining algorithms that run on top of Apache Spark. Apache Spark is a framework that processes big data loading it into a cluster's memory and query it repeatedly. Some studies2 point out that Spark is a tool 100x faster than Hadoop.

Examples

Recommendation Example

parse_data on “Path/to/DataSet” then
 produce(user_vectors) then
 produce(similarity_matrix) then
 multiply("similarity_matrix" by "user_vectors")then
 produce(recommendation)
 write_on “OutPut/Path” then execute

This is an example of item-based collaborative filtering which is based on the similarity between items calculated using people's ratings of those items. We had just to break each part of the algorithm into pieces of simple code. This way, we could build the item and user matrix in order to multiplicate both of them to generate the recommendation matrix that was written into the output path passed by the user. Documentation

  • parse_data:

    • it takes the path to the user's dataset.

    Example:
    parse_data on ("Path/to/dataSet")

  • write_on:

    • it takes a path in order to write into it an output detemined by the user.

    Example:
    write_on “OutPut/Path”

  • produce:

    • it is used in order to create a matrix that can represent either a user vector, or an item similarity matrix or a recommendation matrix. This way, it can take three different arguments: similarity_matrix/user_vectors/recommendation.

    Example:

    • produce(user_vectors)
    • produce(similarity_matrix)
    • produce(recommendation)

  • multiply:

    • It mutiplies two matrix and returns the produced matrix.

    Example:
    multiply("similarity_matrix"by "user_vectors")

  • then:

    • it is used in order to take the output from the previous method and pass it to a next method.

    Example:
    parse_data on “Path/to/DataSet” then produce(user_vectors)

  • as:

    • it creates a alias that is attached to a object, so that the user can use this object by calling its alias instead of object`s name.

    Example:
    produce(user_vectors as matrix_one)

  • execute:

    • it runs the whole command and that's why it is always the last command written.

    Example:
    parse_data on ("Path/to/dataSet") the produce (user_vectors) then execute

Performance

We executed a set of experiments in order to compare the execution time of our DSL to Mahout-Spark. Our goal was to find out whether or not our implementation influenced the execution time negatively. In order to make the comparasion, we executed the item-based collaborative filtering algorithm to build the similarity matrix. This way, we have used a computer cluster with zero (localhost), one, two and three slaves along with four different inputs: 10MB, 25MB, 50MB, 100MB. Each cluster node has the following configuration:

  • 320GB de HD
  • 8 cores
  • 8GB de Ram
